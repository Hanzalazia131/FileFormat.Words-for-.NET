Word Document C# API System Outlines

The following outlines provide an overview of the internal namespaces and classes, including their complexities. While the internal structure is intricate, the public APIs are designed to be straightforward and user-friendly.

Please refer to the API Reference to explore the public interfaces in detail.

FileFormat.Words Namespace

  • Document Class
    • Provides the primary interface for users to create, load, and modify Word documents.
    • Acts as a facade for interaction with internal classes in the OpenXML.Words, OpenXML.Words.Data, and OpenXML.Templates namespaces.

FileFormat.Words.IElements (Custom Objects)

  • Custom elements, including Paragraphs, Images, Tables, etc., that mirror the structure of a Word document.
  • Serve as a data structure for transferring data to and from OpenXML objects.
  • Provides a user-friendly interface to API users for interacting with Word document content.

OpenXML.Words Namespace

  • Document Class (Internal)
    • Provides operations for loading an existing Word document into OpenXML and creating a new Word document from scratch.
    • Serves as a bridge between the custom document elements in FileFormat.Words and the OpenXML-based Word document.
    • Sets the WordProcessing package for OpenXML.Words.Data.OOXMLDocData to enable synchronization.
    • Utilizes templates from the OpenXML.Templates namespace when creating new documents.

OpenXML.Words.Data Namespace

  • OOXMLDocData Class (Internal, Static)
    • Performs static operations to insert, update, and remove elements within the OpenXML-based Word document.
    • Receives synchronization instructions from the FileFormat.Words.Document class.
    • Ensures changes made to custom objects are reflected in the OpenXML document.

OpenXML.Templates Namespace

  • Provides pre-defined templates created using OpenXML SDK Productivity Tools.
  • Includes classes for core properties and custom properties, offering document metadata and customization capabilities.
  • Templates are utilized when creating new Word documents from scratch within the OpenXML.Words.Document class.

This extended architecture encompasses the use of templates and properties from the OpenXML.Templates namespace, enriching the document creation process by providing pre-defined structures and metadata customization options for newly created documents. The FileFormat.Words.Document class remains the central interface for users, orchestrating interactions with various internal components from multiple namespaces.
